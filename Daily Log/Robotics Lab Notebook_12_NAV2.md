# Part 5. Autonomous Navigation using Navigation2

## Objective

After generating and saving an occupancy grid map using SLAM Toolbox, the next step is enabling autonomous navigation.

In this section, Navigation2 is integrated with TurtleBot3 to localize the robot on the previously generated map, generate a collision-free path, and autonomously navigate to a target goal.

앞선 파트에서는 SLAM Toolbox를 이용하여 Occupancy Grid Map을 생성하고 저장하였다.

이번 파트에서는 저장된 지도를 이용하여 Navigation2를 실행하고, Robot의 현재 위치를 추정한 뒤 목표 지점까지의 경로를 생성하여 자율주행을 수행한다.

---

# 1. What is Navigation2?

Navigation2 (Nav2) is the autonomous navigation framework for ROS2.

It enables a mobile robot to localize itself, plan a path, avoid obstacles, and navigate to a specified goal using a previously generated map.

Navigation2(Nav2)는 ROS2에서 사용하는 자율주행 프레임워크이다.

저장된 지도를 기반으로 Robot의 위치를 추정하고, 경로를 생성하며, 장애물을 회피하면서 목표 지점까지 이동하는 기능을 제공한다.

---

# 2. Why is Navigation2 Needed?

SLAM creates a map of the environment but does not move the robot autonomously.

Navigation2 uses the generated map to estimate the robot pose, compute an optimal path, and continuously control the robot until it reaches the goal.

SLAM은 주변 환경의 지도를 생성하지만 Robot을 목적지까지 이동시키지는 않는다.

Navigation2는 생성된 지도를 이용하여 Robot의 위치를 추정하고 최적의 경로를 생성한 뒤, 목적지까지 지속적으로 Robot을 제어한다.

---

# 3. Navigation2 Input and Output

Navigation2 receives sensor data together with the previously generated map and produces navigation commands for the robot.

Navigation2는 저장된 지도와 센서 데이터를 입력으로 받아 Robot의 자율주행을 위한 제어 명령을 생성한다.

### Input

- Saved Map (`my_map.yaml`)
- LaserScan (`/scan`)
- Odometry (`/odom`)
- TF (`/tf`)
- Navigation Goal

### Output

- Robot Pose (`/amcl_pose`)
- Global Path (`/plan`)
- Velocity Command (`/cmd_vel`)
- TF (`map → odom`)

---

# 4. Launching Navigation2

Navigation2 was launched using the occupancy grid map generated by SLAM Toolbox.

SLAM Toolbox에서 생성한 Occupancy Grid Map을 이용하여 Navigation2를 실행하였다.

```bash
export TURTLEBOT3_MODEL=burger

ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py

ros2 launch turtlebot3_navigation2 navigation2.launch.py \
map:=$HOME/my_map.yaml
```

---

# 5. Navigation2 Architecture

Navigation2 consists of multiple servers that cooperate to perform autonomous navigation.

Each server is responsible for a specific task such as localization, path planning, path following, or behavior management.

Navigation2는 여러 개의 Server(Node)로 구성되어 있으며 각 Node는 자율주행의 특정 기능을 담당한다.

각 Server가 서로 협력하여 Localization부터 Goal 도착까지 전체 Navigation 과정을 수행한다.

```
                Navigation Goal
                       │
                       ▼
                 BT Navigator
                       │
                       ▼
                Planner Server
                 ▲           │
                 │           ▼
           Robot Pose    Global Path
                 ▲           │
                 │           ▼
               AMCL    Controller Server ◄──── Local Costmap
                 ▲           │                     ▲
                 │           ▼                     │
            Map Server    cmd_vel           LaserScan/Odom
                 ▲           │                     │
                 │           ▼                     │
            Saved Map    TurtleBot3 ───────────────┘


```

Navigation2 is composed of multiple independent servers that exchange information through ROS2 Topics, TF, and Actions.

Navigation2는 여러 개의 독립적인 Server가 ROS2 Topic, TF, Action을 이용하여 정보를 주고받으며 동작한다.

---

# 6. Core Components of Navigation2

Navigation2 consists of several core components.

Navigation2는 여러 개의 핵심 구성 요소로 이루어진다.

### AMCL

AMCL estimates the robot pose using the saved occupancy grid map and LaserScan.

AMCL은 저장된 지도와 LaserScan을 이용하여 Robot의 현재 위치를 추정한다.

---

### Planner Server

Planner Server computes the collision-free global path from the current robot pose to the navigation goal.

Planner Server는 현재 Robot 위치에서 Goal까지의 충돌 없는 Global Path를 생성한다.

---

### Controller Server

Controller Server follows the generated global path and continuously publishes velocity commands.

Controller Server는 생성된 Global Path를 따라 Robot이 이동할 수 있도록 속도 명령을 지속적으로 생성한다.

---

### BT Navigator

BT Navigator manages the entire navigation process using a Behavior Tree.

BT Navigator는 Behavior Tree를 이용하여 Navigation 전체 과정을 제어한다.

It requests path planning, starts path following, and performs recovery behaviors when necessary.

Planner와 Controller를 호출하며, 필요 시 Recovery Behavior도 수행한다.

---

### Global Costmap

The Global Costmap represents the traversable area of the entire environment.

Global Costmap은 전체 환경에서 이동 가능한 영역과 장애물을 표현한다.

It is mainly used by the Planner Server.

주로 Planner Server가 Global Path를 생성할 때 사용된다.

---

### Local Costmap

The Local Costmap represents nearby obstacles around the robot.

Local Costmap은 Robot 주변의 장애물을 실시간으로 표현한다.

It is mainly used by the Controller Server for local obstacle avoidance.

주로 Controller Server가 장애물을 회피하며 Robot을 제어할 때 사용된다.
